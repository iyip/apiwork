/features

A collection of features supported by answerx and visible to the authenticated user. The entire feature set will have a status, which can be either "READY" or "UPDATING". When the status is ready, it means there is no pending changes for any feature in the entire feature set which is in the process of propagating to edge hosts. When the status is "UPDATING", it means there is pending changes of some/all features that are propagating to edge hosts. In such situation, there will also be a jobHref link present in the json response that displays job completion status.

/features get put

get

Get the collection of features supported by answerx and visible to the authenticated user. For beta, if details param is true and the feature is ECS, then there will also be a config attribute present that includes ECS detailed configuration info. If details param is false, then the ECS configuration info will not be returned, and client has to pull the feature detail url to get such info.

put

Updates the status of the features in the collection, client will only send the features that have changed (either changed in state or in case of ECS, configuration). Since feature update takes a while to get propogated to all edge hosts, server will return 202 accepted http status code, and the response contains the job link and resource link. percentComplete will either be 0 or 100 for beta, and message will contain the reason for failure should the job failed.

/reports/totalReq get

get

Report for total DNS requests, all or break down by region, for a given time period. Report json structure is - timeSeries is an array. Each array element is an object with timestamp and metrics. Timestamp is epoch time, and metrics is an array. Metrics array has 2 elements (corresponding to metrics type), each element is an object with type and items. Type can be 'all' or 'region'. When type is 'all', items array has one element (name is 'All', value is number). When type is 'region', items array has n elements (name is 'NewYork/Boston/DC/LA/…', value is corresponding number)
