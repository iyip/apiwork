/features

A collection of features supported by answerx and visible to the authenticated user. The entire feature set will have a status, which can be either "READY" or "UPDATING". When the status is ready, it means there is no pending changes for any feature in the entire feature set which is in the process of propagating to edge hosts. When the status is "UPDATING", it means there is pending changes of some/all features that are ropagating to edge hosts to edge hosts. In such situation, there will also be a jobHref link present in the json response that displays job completion status.

/features get put

get

Get the collection of features supported by answerx and visible to the authenticated user. For beta, if details param is true and the feature is ECS, then there will also be a config attribute present that includes ECS detailed configuration info. If details param is false, then the ECS configuration info will not be returned, and client has to pull the feature detail url to get such info.

put

Updates the status of the features in the collection, client will only send the features that have changed (either changed in state or in case of ECS, configuration). Since feature update takes a while to get propogated to all edge hosts, server will return 202 accepted http status code, and the response contains the job link and resource link. percentComplete will either be 0 or 100 for beta, and message will contain the reason for failure should the job failed.

/reports/dnsReq get

get

Report for total DNS requests, all or break down by region, for a given time period. Report json structure is - within metrics attribute is a collection of type and items array. Type can be All or Region, which show all dns requests, or dns requests broken down by region respectively. Within items is a collection of name and timeSeries array. name can be all or region name. Each item in timeSeries has timestamp and value.

/admin/account/{accountId}/ get patch

get

Get the details of a specific account, identified by its id. If the configured flag of the account is false, then upon initial access, user must go through the setup process to provide the anycastIP and customerId, before he can access the system. If the configured flag is true, then the account has already been setup (one time process).

Update a specific account, identified by its id.
